# Changelog — pattern-language

Newest first. The current version is also noted at the foot of SKILL.md.

*v0.7 (July 2026) — **typed evidence model**, replacing the v0.6.1 known-use/citation dichotomy after the fourth baseline run (Sonnet × Shadowrun) exposed the remaining gap: a strong model promoted five patterns to ✻✻ on genre convention, disclosing the substitution in the pathfinder — the skill had never defined what evidence *means* in a domain without fielded practice. `pattern-form.md` rule 6 now defines three tiers: **instances** (specific, nameable, dateable, in the language's declared application domain — the only tier that lives in Known uses and the only tier that promotes confidence), **designed-system corroboration** (body prose; never promotes alone), and **narrative convention** (Problem-body argument at most; never promotes — fiction is a biased simulator). Confidence calibration is purpose-relative: the pathfinder's Evidence section must declare what counts as an instance for *this* language (play-facing → documented campaigns/sessions; fiction-authoring → specific named works can qualify); improvised per-pattern semantics are a HIGH qualitative finding. No blanket zero for fictional domains — a pattern documented across three independent campaigns earns ✻✻ normally. Validator: new heuristic WARN when a starred pattern's Known uses use vague-plural/corroboration vocabulary (nameable-and-dateable is the instance test); regression-tested against run 4's failure signature. `examples/garden-crew`: Bed Stewards' Tier-B CODEOWNERS item moved to body prose as modelled corroboration, replaced by a third genuine instance.*
*v0.6.2 (July 2026) — `validation.md` qualitative checks hardened after a second Haiku baseline (setting up a Shadowrun crew) replicated two soft-fence failures across domains. **Name test** now names the two observed failure modes explicitly — imperative-verb names ("Recover from Failure") and abstract-concept names ("Reputation Economy") — while declining a mechanical verb-lint (false-positives on APL gerund names). **Gradient coherence** now checks both single-axis-no-drift and label-matches-structure (weak models build a formation-chronology then mislabel it "organizational scope"). Both stay qualitative fences the validator cannot automate.*
*v0.6.1 (July 2026) — `pattern-form.md`: sharpened the definition of a **known use** after a Haiku baseline run filled every known-uses section with adjacent-domain citations ("MechWarrior 5 tracks reputation") rather than independent instances of the pattern. A known use is now explicitly distinguished from a citation — corroboration belongs in body prose or the pathfinder landscape, not the evidence slot, and cannot bear rule-of-three weight. Surfaced by the first skill-under-weak-model baseline; the validator cannot police known-use quality, only presence, so this is a qualitative fence.*
*v0.6 (July 2026) — assurance and process layer. Validator: fixed a reciprocity blind spot dating from v0.3 (a `t > n` guard silently passed edges asserted only via `links_up` — 558 of APL's 1,758 edges; one-way edges now warn from the asserting side, 1,251 on APL); `section()` and the context split are now case-insensitive ('## Known Uses' no longer false-warns). New **Mine** operation with `references/mining.md` (intake-briefed harvest → cluster-by-tension → forces-test → route pipeline; internal sources yield candidates, public sources corroboration; provenance and null results written back to the pathfinder with resumption markers for re-entrant passes). New SKILL.md § Context discipline & model weight (working-set rule, batch–validate–drop, operation/tier envelope). New `examples/garden-crew` worked language (4 patterns + pathfinder + sequence; validates 0/0) doubling as the clean test fixture. New `scripts/tests/test_validator.py` (8 groups, 16 checks) wired into the packaging workflow as a gate. Writers'-workshop protocol added to `references/validation.md`. Wording sweep: pathfinder heading order is convention not enforcement; folder shape recommended at any size; orphan semantics documented.*
*v0.5 (July 2026) — exemplar-grounded update from a comparative read of scrumbook.org (A Scrum Book), apl-md (the full 253-pattern corpus run through this validator: 0 FAILs, 693 reciprocity warns, ~29% of edges mutual, ~14 links/pattern), and Group Works. New: the **pathfinder** (`references/pathfinder.md`, `PATHFINDER.md` / `## Pathfinder` — purpose, scope, declared gradient, landscape, evidence, growth queue; supersedes the optional README); the **Apply** operation with sequences as first-class artifacts (purpose, entry conditions, steps, cautions; validated links); optional **Known uses** (rule of three, tied to ✻✻) and **Cautions** pattern sections; **gradient** doctrine generalised beyond spatial scale to any declared monotone axis; optional `###` clusters in the index (APL's own two-level structure); validator reporting grouped by category with per-category cap, `--verbose`, and a network-statistics line.*
*v0.4 (June 2026) — drawn from the Engine 52 worked language. Two delivery shapes are now first-class: the file-per-pattern folder (default) and a single wiki-doc whose embedded `mermaid graph TD` block is the canonical link source. The validator auto-detects shape from its path and gained a single-doc mode (header/scale/confidence + Mermaid relations + problem/therefore presence) and BOM-tolerant reads. Lateral links — `sympathies:` / `tensions:` (mutual; rendered `-.-` and `x--x`) — are now part of the schema and checked in both shapes. Folder checks from v0.3 are unchanged.*
*v0.3 (June 2026) — validator: `#`-in-name check moved to correct field; reciprocity WARNs halved (one per broken pair); context split uses regex matching `section()` so `## Problem:` is handled correctly; unknown frontmatter keys emit WARN (catches typos); misleading `prose_mentions` comment removed. Docs: `language-structure.md` index rule aligned with `validation.md`; reporting-format example updated to match script output. Cluster: complements skill-architecture/skill-creator; no dependencies on other skills.*
